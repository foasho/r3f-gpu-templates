# Skill作成コマンド

あなたはClaude Codeの設定に詳しいエンジニアです。
ユーザーと対話しながら、適切なSkillを作成してください。

---

## Skillとは

Skillは、Claude Codeに特定の専門知識やワークフローを追加するための機能です。
プロジェクト固有のルール、手順、ベストプラクティスを定義し、チームで共有できます。

### Skillの特徴

- **再利用可能**: 一度定義すれば何度でも参照可能
- **共有可能**: チームメンバー間で知識を共有
- **文脈提供**: Claudeに特定のコンテキストや制約を伝える
- **ワークフロー定義**: 複数ステップの手順を標準化

### Skillの用途例

| カテゴリ | 例 |
|---------|-----|
| コーディング規約 | 命名規則、ファイル構成、コメントスタイル |
| アーキテクチャ | 設計パターン、ディレクトリ構造、モジュール分割 |
| ワークフロー | デプロイ手順、レビュープロセス、テスト方法 |
| ドメイン知識 | ビジネスルール、用語集、業務フロー |
| セキュリティ | 認証・認可ルール、データ取り扱い規則 |

## 作成手順

### Step 1: 要件の確認

ユーザーに以下を質問してください：

1. **目的**: どのような知識/ワークフローを定義しますか？
   - コーディング規約
   - 設計パターン
   - 作業手順
   - ドメイン知識
   - その他

2. **対象**: 誰が/どの場面で使いますか？
   - 全ての開発者
   - 特定のロール（フロントエンド、バックエンド等）
   - 特定のタスク（レビュー、デプロイ等）

3. **内容**: 具体的に何を含めますか？
   - ルールや制約
   - 手順やステップ
   - 例やテンプレート
   - 参考資料へのリンク

4. **適用範囲**: いつ参照されるべきですか？
   - 常に（CLAUDE.mdに含める）
   - 必要に応じて（/skillコマンドで呼び出し）

### Step 2: Skill内容の作成

ユーザーの回答に基づいて、以下の形式でSkillを作成します。

**.claude/skills/<skill名>.md ファイル形式:**

```markdown
---
name: <Skill名>
description: <Skillの概要>
version: "1.0.0"
---

# <Skill名>

## 概要
<このSkillの目的と適用場面>

## ルール/手順
<具体的なルールや手順>

## 例
<具体例やテンプレート>

## 参考
<関連ドキュメントやリソース>
```

### Step 3: 確認と保存

1. 作成したSkillをユーザーに提示
2. 修正点があれば調整
3. 承認後、`.claude/skills/<skill名>.md` に保存

## Skillテンプレート例

### 例1: コーディング規約

```markdown
---
name: coding-standards
description: プロジェクトのコーディング規約
version: "1.0.0"
---

# コーディング規約

## 概要
このドキュメントはプロジェクト全体で適用されるコーディング規約を定義します。

## 命名規則

### 変数・関数
- 変数名: camelCase（例: `userName`, `isActive`）
- 関数名: camelCase、動詞で始める（例: `getUserName`, `validateInput`）
- 定数: UPPER_SNAKE_CASE（例: `MAX_RETRY_COUNT`）

### ファイル・ディレクトリ
- コンポーネント: PascalCase（例: `UserProfile.tsx`）
- ユーティリティ: camelCase（例: `formatDate.ts`）
- テスト: `*.test.ts` または `*.spec.ts`

## コメント
- 複雑なロジックには必ずコメントを付ける
- JSDoc形式で関数を文書化する
- TODO/FIXMEには担当者と日付を含める

## 禁止事項
- any型の使用（型安全性のため）
- console.logの本番コードへのコミット
- マジックナンバーの直接使用
```

### 例2: デプロイ手順

```markdown
---
name: deploy-procedure
description: 本番環境へのデプロイ手順
version: "1.0.0"
---

# デプロイ手順

## 概要
本番環境へのデプロイ時に従う標準手順です。

## 前提条件
- [ ] 全てのテストがパスしている
- [ ] コードレビューが承認されている
- [ ] CHANGELOGが更新されている

## 手順

### 1. リリースブランチの作成
```bash
git checkout main
git pull origin main
git checkout -b release/v<version>
```

### 2. バージョン更新
```bash
npm version <major|minor|patch>
```

### 3. デプロイ実行
```bash
npm run deploy:production
```

### 4. 動作確認
- [ ] ヘルスチェックエンドポイントの確認
- [ ] 主要機能の動作確認
- [ ] エラーログの監視

## ロールバック手順
問題発生時は以下を実行：
```bash
npm run rollback:production
```
```

### 例3: ドメイン知識

```markdown
---
name: domain-glossary
description: プロジェクトのドメイン用語集
version: "1.0.0"
---

# ドメイン用語集

## 概要
このプロジェクトで使用される業務用語の定義です。

## 用語

### ユーザー関連
- **アクティブユーザー**: 過去30日以内にログインしたユーザー
- **プレミアムユーザー**: 有料プランに加入しているユーザー

### 注文関連
- **カート**: 購入前の商品一時保存領域
- **チェックアウト**: 購入手続きのプロセス
- **確定注文**: 支払い完了した注文

## ビジネスルール
- 注文確定後24時間以内はキャンセル可能
- プレミアムユーザーは送料無料
```

## 注意事項

- Skillはプロジェクト固有の知識を蓄積する場所です
- 汎用的な内容はCLAUDE.mdに、詳細な内容はSkillに分離すると管理しやすい
- バージョン管理して変更履歴を追跡しましょう
- 定期的にレビューして最新の状態を維持しましょう
